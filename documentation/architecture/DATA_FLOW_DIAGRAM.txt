================================================================================
                         KEY DECISION POINTS
================================================================================

1. TEMPLATE ENGINE: Jinja2 ✓
   ├─ Rationale: Best Python integration, clean syntax, large ecosystem
   ├─ Alternative: Mako (too complex), Chameleon (smaller ecosystem)
   └─ Dependency: jinja2>=3.0.0 in requirements.txt

2. DEPLOYMENT STRATEGY: /docs folder ✓
   ├─ Rationale: Simpler, version-controlled, no branch management
   ├─ Alternative: gh-pages branch (complex, harder to maintain)
   └─ GitHub Pages Setting: Settings > Pages > Deploy from /docs

3. DATA FLOW: Models → JSON → HTML ✓
   ├─ Separation of concerns: Serialization, rendering, orchestration
   ├─ Serializer: DashboardDataExporter (models → JSON)
   ├─ Renderer: DashboardRenderer (JSON → HTML via Jinja2)
   └─ Orchestrator: DashboardBuilder (coordinates entire pipeline)

4. WORKFLOW INTEGRATION: Minimal changes ✓
   ├─ New step: DashboardBuilder.build() after stats generation
   ├─ New CLI command: spark dashboard
   ├─ No breaking changes to existing workflow
   └─ Optional dashboard generation via report_mode input

5. ASSET MANAGEMENT: Copy & optimize ✓
   ├─ CSS: Copy from src/spark/static/ → docs/assets/css/
   ├─ JS: Copy from src/spark/static/ → docs/assets/js/
   ├─ SVG: Copy from output/ → docs/assets/images/svg/ + optimize
   └─ Minification: Reduce payload size
